# A股巴菲特安全边际法量化筛选

## 用途

每个交易日收盘后，对 **A 股全市场（所有个股，不限定指数成份）** 执行巴菲特价值投资风格的量化初筛与六维深度评分，估算安全边际率，经三重复核后，生成 **HTML 表格** 与 **Excel（.xlsx）** 两种格式完全一致的评估结果。

> 方法论严格依据《巴菲特安全边际法 — 量化评估框架》（维度权重、评分档位、决策矩阵、内在价值估算均以该框架为准）。

## 触发词

- "A股巴菲特筛选" / "巴菲特安全边际法每日筛选"
- "安全边际法筛选" / "按巴菲特标准筛A股"
- "价值投资候选池" / "生成巴菲特式选股报告"
- "用巴菲特方法找低估股票" / 类似表述

## 运行参数（环境变量，可选）

| 变量 | 说明 | 默认 |
|------|------|------|
| `TRADE_DATE` | 交易日 YYYY-MM-DD，使用当日收盘数据 | 今天 |
| `OUTPUT_DIR` | 输出目录 | `<当前工作区>/outputs` |
| `UNIVERSE_LIMIT` | 全市场拉取上限（防止截断） | `6000` |

输出文件名：
- `OUTPUT_DIR/巴菲特筛选_YYYY-MM-DD.html`
- `OUTPUT_DIR/巴菲特筛选_YYYY-MM-DD.xlsx`

## 数据源优先级

1. **westock-data**（`data_finance` / `data_profile` / `data_quote` / `data_sector` / `data_risk` / `data_notice` / `data_events` / `data_report`）——结构化数据主力。
2. **westock-tool**（`tool_filter` / `tool_ranking`）——量化初筛与排行。
3. 数据不足时降级至 **a-stock-data** skill 或 **tdx-connector** 补充。

> 行情用当日收盘数据；财务用最新已披露季报/年报。

## 执行流程（五阶段，逐步验证，不得跳过）

### 阶段零（前置·必须最先执行）：交易日历校验

> 本关卡在所有取数动作之前执行，目的是让自动化在周末/法定节假日静默跳过，避免对休市日无效运行。

1. 确定待校验日期 `CHECK_DATE = TRADE_DATE`（默认今天，格式 `YYYY-MM-DD`）。
2. 调用交易日历工具：
   - 工具：`mcp__westock-mcp__data_trade_calendar`
   - 参数：`{ "date": "<CHECK_DATE>" }`（单日查询，无需 `trading_only`）
3. 解析返回：`data.days[0]`
   - **若 `days` 为空，或 `days[0].isTrading === false`** → 判定为**非交易日**（周末/休市）→ **立即优雅终止**：
     - 不拉取任何行情/财务数据，不生成 HTML/XLSX 文件。
     - 输出一行日志，如：「<CHECK_DATE> 非交易日（周末/法定节假日），跳过本次筛选。」
     - 结束流程（对自动化而言即静默退出，不报错）。
   - **若 `days[0].isTrading === true`** → 判定为交易日 → 继续下一阶段。

> 依据：实测 `data_trade_calendar` 单日查询对交易日返回 `isTrading:true`、对周六返回 `isTrading:false`，结构稳定可依赖。

### 阶段〇：准备工作

1. 确定 `TRADE_DATE`（默认今天）。
2. 创建 `OUTPUT_DIR`（不存在则建）。
3. 加载框架评分标准（维度权重与分档，见下文「评分标准速查」）。
4. 建立工作 JSON：`OUTPUT_DIR/_raw_<TRADE_DATE>.json`，逐步写入每只候选股的六维原始数据与评分，供渲染脚本消费。

### 第一阶段：全量初筛（一票否决制）

目标：对 A 股全市场执行 8 条一票否决过滤，统计 **全市场总数 / 各门槛排除数 / 通过候选数**。

**0. 获取全市场 universe**
- 用 `tool_filter`（market=`hs`，`expression` 取宽松条件如 `PE_TTM > 0`，`limit`=`UNIVERSE_LIMIT`）拉取全量 A 股代码列表，记为 `UNIVERSE`，记录总数。
- 若 `tool_filter` 无法稳定返回全量，改用 `data_stocklist`（mode=rank）或 `data_search` 补全。

**1~8. 八条一票否决**（任一不满足即排除）

| # | 否决项 | 可用工具与字段 | 备注 |
|---|---------|----------------|------|
| 1 | 上市时间 < 3 年 | `data_profile`（上市日期）或 `tool_filter` 表达式 | 以 `TRADE_DATE` 回溯 3 年 |
| 2 | 近 3 年中有 2 年净利润为负 | `data_finance`（利润表，num≥3） | 统计近 3 个年报净利润为负年数 ≥ 2 |
| 3 | ROE 连续 3 年 < 8% | `data_finance`（num≥3）或 `tool_filter` 表达式 `ROE > 8` | 连续 3 年 < 8% 排除 |
| 4 | 资产负债率 > 70%（金融业除外） | `data_finance`（资产负债表）或 `tool_filter` `max_debt_ratio=70` | 金融业（银行/保险/券商）豁免 |
| 5 | 审计意见为非标准无保留 | `data_finance` 审计意见字段 / `data_risk`(st) | 无字段则标注「数据缺失」 |
| 6 | 近 1 年有证监会立案调查或重大违规 | `data_risk`(lawsuit) / `data_notice`（违规类公告） | 近 1 年窗口 |
| 7 | 商誉 / 净资产 > 50% | `data_finance`（商誉、净资产） | 比值 > 50% 排除 |
| 8 | 大股东质押比例 > 60% | `data_risk`(pledge) | 无数据标注「数据缺失」 |

**执行策略**
- 量化可筛项（1、3、4 的财务部分）优先用 `tool_filter` 的 `expression` + `intersect` 一次性求交集，拿到初步通过集。
- 非直接可筛项（2、5、6、7、8 及审计意见）对通过集（或抽样）用 `data_finance` / `data_risk` / `data_notice` **逐项补充**。
- 每项统计排除数量；无法获取的标「数据缺失」，**不得忽略**。

**阶段产出（写入 JSON 与最终文件备注区）**
- 全市场总数 `UNIVERSE_COUNT`
- 各条门槛分别排除数 `excluded_by[1..8]`
- 通过初筛候选数 `candidates_count`

### 第二阶段：六维深度量化评分

对每只候选股，用 `data_finance` / `data_profile` / `data_sector` / `data_quote` / `data_report` 逐项取数，按下表打分（2/5/8/10 分制）。

#### 评分标准速查（来自框架）

**维度1 盈利能力 20%** — 近 5 年 ROE（data_finance）
- ROE ≥ 20%（连续5年）→ 10；15~20%（连续5年）→ 8；10~15%（连续3年）→ 5；< 10% 或波动剧烈 → 2
- 辅助：ROA ≥ 7%、ROIC ≥ 12%

**维度2 护城河 18%** — 近 5 年毛利率（data_finance）
- 毛利率 ≥ 40%（稳定）→ 10；30~40% → 8；20~30% → 5；< 20% 或趋势下行 → 2
- 定性（data_profile / data_report）：品牌、转换成本、网络效应

**维度3 估值合理 25%** — PE(TTM) 与行业均值（data_quote + data_sector）
- PE < 行业均值 × 0.7 → 10；< 0.9 → 8；接近均值 → 5；显著高于 → 2
- 辅助：PEG = PE / 近3年净利润复合增长率，PEG ≤ 1.0 正向调整

**维度4 财务稳健 17%** — 资产负债率、流动比率（data_finance）
- 资产负债率 < 30% 且 流动比 > 2 → 10；30~50% → 8；50~65% → 5；> 65% → 2

**维度5 盈利质量 15%** — 近 3 年经营现金流 / 净利润（data_finance）
- 比值 ≥ 1.2 → 10；1.0~1.2 → 8；0.7~1.0 → 5；< 0.7 → 2
- 辅助：自由现金流是否连续为正

**维度6 补充因子 5%（加分/减分制）**
- 股息率 ≥ 3% 且持续分红 → +1
- 近 3 年营收复合增长率 ≥ 10% → +1（data_finance）
- 管理层增持 / 公司回购（data_events / data_notice）→ +1
- 应收账款激增 / 商誉过高 → -1

**加权总分** = Σ(各维度得分 × 权重)，满分 10 分（维度6 加减分在总分外单独列出「补充因子加分」列，并计入上限保护）。

每只候选股原始数据与各维度分数写入 `_raw_<TRADE_DATE>.json`。

### 第三阶段：安全边际率估算

对 **加权总分 ≥ 6.0** 的个股计算安全边际率。

1. **快速内在 PE 法估算每股内在价值**
   - 内在 PE（依据框架"快速内在PE估值法"）：高 ROE(>20%)+强护城河 → 22.5×；中高 ROE(15~20%) → 17.5×；中等 ROE(10~15%) → 12.5×；低 ROE/高杠杆 → 不估算（标注「不适用」）。
   - 归一化 EPS = 近 3 年扣非 EPS 平均值（data_finance）。
   - 每股内在价值 = 内在 PE × 归一化 EPS。
2. **安全边际率** = (每股内在价值 − 当前股价) / 每股内在价值 × 100%（当前股价取自 `data_quote` 收盘价）。
3. 总分 < 6.0 者不估算，安全边际率与内在价值列留空并标注。

---

### 第四阶段：三重复核（严格执行）

**第一重 · 数据完整性**
- 逐一确认每只候选股每个维度**至少 3 个独立数据源**。
- 仅单一数据源 → 标注「数据源不足，评分可信度降低」。

**第二重 · 评分一致性**
- 随机抽取 **30%** 候选股（不少于 3 只），用原始数据**手动重算**加权总分。
- 差异 > 1 分 → 修正后对**全部**候选股重算。

**第三重 · 决策合理性**
- 对总分 ≥ 7.0 的"买入/重仓"候选股，反向论证：「如果这只股票不值得买，最可能的三个理由是什么？」
- 在最终表格新增「**风险提示**」列，写入反方观点。

**输出列清单（顺序固定）**
序号 | 股票代码 | 股票名称 | 所属行业/板块 | ROE(5年均值) | 毛利率(5年均值) | PE(TTM) | PEG | 资产负债率 | 经营现金流/净利润 | 盈利能力得分 | 护城河得分 | 估值合理得分 | 财务稳健得分 | 盈利质量得分 | 补充因子加分 | 加权总分 | 内在价值估算(元) | 当前股价(元) | 安全边际率 | 操作建议 | 风险提示 | 数据完整度

**操作建议映射（框架决策矩阵）**
- 加权总分 ≥ 8.0 且 安全边际率 ≥ 35% → 重仓买入
- 7.0~7.9 且 25%~35% → 分批买入
- 6.0~6.9 且 20%~25% → 观察仓
- 5.0~5.9 且 10%~20% → 加入自选
- < 5.0 或 < 10% → 放弃

**Excel 格式要求**（脚本实现）
- 表头行：加粗、浅灰底色、冻结首行。
- 数字列 2 位小数；百分比列百分比格式。
- 评分列（2/5/8/10）居中对齐；操作建议列着色（绿=买入/分批、橙=观察/自选、红=放弃）。
- 列宽自适应；数据区域加边框。
- 「汇总」工作表：筛选日期、全市场总数、初筛通过数、最终分析数、各行业分布、TOP10 排行。

**HTML 格式要求**
- 页面头部为摘要区（同 Excel 汇总）。
- 表格下方备注：三重复核执行状态（时间戳+每轮摘要）、数据缺失项说明、免责声明。

**容错**
- 单只股票数据获取失败 → 跳过并记录到备注区，**不中断**整体流程。
- 结果生成后，用 `present_files` **同时展示 HTML 与 Excel**。

## 已知数据限制（透明标注）

- 审计意见标准无保留通常无直接 API，用 ST/风险标签代理；确实无字段则标「数据缺失」。
- 大股东质押比例接口返回有限，缺失时标「数据缺失」而非忽略。
- 扣非 EPS 以基本每股收益近似（注明）。
- 全市场逐只补充调用量较大，建议以自动化（每日收盘后）触发本技能。
